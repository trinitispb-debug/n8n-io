## 🚀 Быстрая настройка в n8n

### Шаг 1: Импорт workflow
1. Откройте ваш n8n instance
2. Нажмите "+" → "Import from file"
3. Выберите файл `telegram-openrouter-bot.json`
4. Нажмите "Import"

### Шаг 2: Настройка Telegram Credentials
1. Перейдите в **Settings** → **Credentials**
2. Нажмите **"Create New"**
3. Выберите тип **"Telegram"**
4. Заполните поля:
   - **Name**: `Telegram Bot Credentials`
   - **Access Token**: `8261495015:AAFryYGymGk3Mmp6OsvuL-mbKIcUVAqIMgQ`
5. Нажмите **"Save"**

### Шаг 3: Получение OpenRouter API Key
1. Зайдите на https://openrouter.ai/
2. Зарегистрируйтесь или войдите в аккаунт
3. Перейдите в раздел **"Keys"**
4. Создайте новый API ключ
5. Скопируйте ключ (он выглядит как `sk-or-v1-...`)

### Шаг 4: Настройка OpenRouter Credentials
1. В разделе **Credentials** создайте новые credentials
2. Выберите тип **"HTTP Header Auth"**
3. Заполните поля:
   - **Name**: `OpenRouter API`
   - **Header Name**: `Authorization`
   - **Header Value**: `Bearer ВАШ_OPENROUTER_API_KEY`
4. Нажмите **"Save"**

### Шаг 5: Подключение credentials к узлам
1. Откройте импортированный workflow
2. В узле **"Telegram Trigger"**:
   - Выберите созданные Telegram credentials
   - Убедитесь, что выбраны updates: `message` и `callback_query`
3. Во всех узлах **"Telegram"** (отправка сообщений):
   - Выберите те же Telegram credentials
4. В узле **"OpenRouter API Request"**:
   - В поле Authentication выберите `Generic Credential Type`
   - Выберите созданные OpenRouter credentials

### Шаг 6: Активация workflow
1. Нажмите кнопку **"Active"** в правом верхнем углу
2. Workflow должен стать активным (зеленая точка)

## 🔧 Возможные проблемы и решения

### Проблема: "Webhook не установлен"
**Решение**: 
1. В узле Telegram Trigger нажмите "Test step"
2. n8n автоматически установит webhook

### Проблема: "Ошибка OpenRouter API"
**Проверьте**:
- Правильность API ключа
- Наличие средств на аккаунте OpenRouter
- Правильность формата Authorization header

### Проблема: "Бот не отвечает"
**Проверьте**:
- Активен ли workflow (зеленая точка)
- Правильность Bot Token
- Логи выполнения в разделе "Executions"
