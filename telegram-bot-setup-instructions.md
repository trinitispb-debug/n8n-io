# Telegram OpenRouter AI Bot - Инструкции по настройке

## Описание
Этот workflow создает Telegram бота, который предлагает различные модели нейросетей через OpenRouter API. Пользователи могут выбирать между различными моделями AI и получать ответы на свои вопросы.

## Возможности бота
- 🤖 Выбор из множества AI моделей (GPT-4, Claude, Llama, Gemini и др.)
- 💬 Интерактивные inline-кнопки для выбора модели
- 📊 Просмотр всех доступных моделей
- 🔄 Легкое переключение между моделями
- ❌ Обработка ошибок API
- 🚀 Поддержка генерации изображений

## Предварительные требования

### 1. Telegram Bot Token
1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Сохраните полученный Bot Token

### 2. OpenRouter API Key
1. Зарегистрируйтесь на [OpenRouter.ai](https://openrouter.ai/)
2. Перейдите в раздел API Keys
3. Создайте новый API ключ
4. Сохраните ключ для настройки

## Настройка в n8n

### 1. Импорт workflow
1. Откройте n8n
2. Нажмите "Import from File"
3. Выберите файл `telegram-openrouter-bot.json`
4. Нажмите "Import"

### 2. Настройка Telegram Credentials
1. Перейдите в Settings → Credentials
2. Создайте новые credentials типа "Telegram"
3. Введите ваш Bot Token
4. Сохраните credentials

### 3. Настройка OpenRouter Credentials
1. В разделе Credentials создайте новые credentials типа "HTTP Header Auth"
2. Назовите их "openrouter"
3. В поле Header Name введите: `Authorization`
4. В поле Header Value введите: `Bearer YOUR_OPENROUTER_API_KEY`
5. Сохраните credentials

### 4. Настройка узлов
1. В узле "Telegram Trigger":
   - Выберите ваши Telegram credentials
   - Убедитесь, что выбран тип обновлений "message"

2. В узле "OpenRouter API Request":
   - Убедитесь, что выбраны credentials "openrouter"
   - Проверьте URL: `https://openrouter.ai/api/v1/chat/completions`

### 5. Активация workflow
1. Нажмите кнопку "Active" в правом верхнем углу
2. Workflow начнет слушать сообщения от Telegram

## Дополнительные настройки

### Добавление новых моделей
1. Откройте узел "Send Welcome Message"
2. В разделе "reply_markup" добавьте новую кнопку:
```json
{
  "text": "🆕 Новая модель",
  "callback_data": "model_provider/model-name"
}
```
3. Обновите узел "Process Model Selection" с описанием новой модели

### Изменение параметров AI
В узле "OpenRouter API Request" можно изменить:
- `max_tokens` - максимальное количество токенов в ответе
- `temperature` - креативность модели (0.0 - 1.0)
- Добавить системный промпт в массив `messages`

### Персистентное хранение выбранной модели
Для сохранения выбранной модели пользователя между сессиями:
1. Добавьте узел базы данных (Redis, PostgreSQL)
2. Сохраняйте `userId` и `selectedModel` при выборе модели
3. Загружайте сохраненную модель в узле "Prepare AI Request"

## Мониторинг и отладка

### Логи
- Используйте узлы "Set" для логирования промежуточных данных
- Проверяйте выполнение workflow в разделе "Executions"

### Обработка ошибок
Workflow включает обработку основных ошибок:
- Ошибки OpenRouter API
- Неправильный формат ответа
- Сетевые ошибки

### Лимиты
- OpenRouter имеет лимиты на количество запросов
- Настройте rate limiting если необходимо
- Мониторьте использование токенов

## Безопасность

### Рекомендации
1. Не делитесь API ключами
2. Используйте переменные окружения для sensitive данных
3. Настройте webhook URL для продакшена
4. Регулярно ротируйте API ключи

### Ограничения доступа
Добавьте проверку пользователей в начале workflow:
```javascript
const allowedUsers = [123456789, 987654321]; // ID разрешенных пользователей
const userId = $json.message.from.id;

if (!allowedUsers.includes(userId)) {
  return { error: "Access denied" };
}
```
